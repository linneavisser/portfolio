Assignment 04.01.01 - A website with a game

Create a responsive website with an animated game using HTML, CSS and JavaScrip.

Process

The pitch for my game was to keep a baby sloth asleep by collecting moons and sheep.

After working on the design I started the coding of the game. I started by learning to use animations and keyframes in CSS. These animations I then added to my Sprites in JavaScript. I used conditional statements in my JavaScript to be able to start, pause, mute etc the game, in which I called on different functions depending on if a condition is true or false.

I used the SMD I made as a blueprint for when coding my JavaScript.

I recorded one of the sounds for the game myself which I manipulated and amplified in Adobe Audition. For the rest of my sounds and background music I used royalty free music that I also edited in Audition.

In the last iteration of the game I made a few changes to design. After playing the games in class I noticed that very few read through long instructions so I shortened them and made it more obvious which were the good and bad elements by adding pictures to my instructions. I rounded the corners of the game screen and instruction screen as well as changde to close button to a back button that matched my other UI-elements for a more cohesive design.

Assignment 04.02.01 - Asset list and design documentation

Create a cohesive design for the game based on general game design rules, document the progress and create a website with all the visual assets.

Process

I created a moodboard with different minimalistic illustrations I was inspired by for the design of my game. I was mainly inspired by Quentin Monge and Tove Janssons drawings, one of which I used to extract my colour palette from. I added different shades of green to add to the feel of the rainforest.

When designing my main character I used circles that I manipulated in different ways to add dimension to the character. To make the user emphasise with my main character more I applied the baby bias when designing it, making its face and eyes unproportionally big for its body.

I kept the bouba/kiki effect in mind when designing both the character and the good and bad elements, making sure that all my good elements were rounded and the bad ones sharp and edgy.

For the background design I used darker colours in the foreground and lighter colours the further away an item is to add dimension.
